Polynomial Solver

Engineered a custom trajectory system using a quadratic function logic:

This ensures the curve is anchored at the gun and target while allowing for smooth, symmetrical offsets.

The Problem

Standard physics updates or simple OverlapSphere checks at line segments can miss thin colliders (Tunneling).

The Solve

Implemented Segmented Raycasting. In the same frame the trajectory is calculated, the system performs discrete Raycasts between each shootingLinePositions[i] and shootingLinePositions[i+1].

Clean Energy

Lighting Decals Optimization Unity
‚ñ∂ Play Game
Goal Create a satisfying "Electrician" fantasy centered on recycling and restoration.
  • Visual Feedback: Engineered a high-contrast lighting loop where the player's progression is immediately visible through the illumination of darkened city districts.
  • Engagement Loop: Balanced the cable-creation economy to reward efficient pathfinding, ensuring the "rehabilitation" of the city felt earned rather than automatic.
  • Impact: Used hit-stops and subtle camera shakes during cable connection to provide a tactile sense of "powering up" the environment.
The Constraint

Real-time lighting for multiple city blocks exceeded mobile GPU budgets. Traditional runtime lightmap switching was rejected to avoid a ~300% increase in build size from redundant texture maps.

The Solve

Developed a Decal-Based Shadow Casting system. Instead of lighting the buildings, I projected large black textures via decals to simulate "unpowered" states.

80% Build Size Reduced vs. multiple lightmap sets
0ms Lighting Overhead SetActive(false) on decal layer
